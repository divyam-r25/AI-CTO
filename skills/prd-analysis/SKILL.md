# PRD Analysis Workflow

## Objectives
- Extract product goals and user personas
- Separate must-have requirements from optional ideas
- Detect hidden constraints: latency, compliance, budget, team size
- Produce assumption log for unknowns

## Required Output
1. Problem statement
2. Target users
3. Must-have features
4. Non-functional requirements
5. Open questions
6. Assumptions register

## Validation Checks
- No missing user persona
- At least one measurable success metric
- All critical constraints explicitly listed
